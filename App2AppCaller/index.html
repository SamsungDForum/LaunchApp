Tizen App to App Caller

Available buttons/actions:

1 - Launch TizenCallee without sending any data to it
2 - Launch TizenCallee and send data to it
0 - Clear logs

Info

This application demonstrates how to perform application launching from an different application.

Main workflow

This application (Tizen App to App Caller) will launch different app (Tizen App to App Callee). Depending on the action chosen by user, application can communicate between themselves or not. When entering Tizen App to App Callee, Tizen App to App Callee application will be moved to background. It will be running, because of the following setting attribute in config.xml file:

background-support="enable"

This means that Tizen App to App Callee will be able to listen to any data incomming from Tizen App to App Callee.

While in Tizen App to App Callee application, press:

  • Return key - to close the application and go back to the Tizen App to App Caller app.
  • Exit key - to exit the application to main SmartTV screen.

No communication

Press 1 to launch Tizen App to App Callee app. In this scenario there will be no communication between apps. Tizen App to App Caller will simply launch the Tizen App to App Callee. Without sending any data. Tizen App to App Callee will not send back any data.

If Tizen App to App Callee was already running before, it will be brought back from background instead of creating a new instance.

With communication

Press 2 to launch Tizen App to App Callee with arguments. Arguments will be sent directly to Tizen App to App Callee. Arguments will be ['apples', 'bananas']. As soon, as Tizen App to App Callee receives the arguments, it will send back an message. The message will be handled by the replyHandler object. If received successful message, it will print it. It should be something like Thank you for the fruits!.

If Tizen App to App Callee was already running in background, it will be killed. A new instance of application will be launched with params provided by Tizen App to App Caller.
